Configuring User Defined Message Search in SAP CPI

Generally, it is not easy to uniquely identify a message required for further analysis when there are many messages in the system. The CPI does not have a function such as User Defined Message Search (UDMS) in PI.

With the help of two standard headers, this feature can be enabled in Integration Flows to define a message based on a unique or key field from the message content.

We may use the Application Message ID and Application MessageType for our purposes. This is achieved by filling the message header ‘SAP_ApplicationID’ and ‘SAP_MessageType’ with an appropriate value.It is on the developer to think what is the appropriate value to place in the Application Message ID. This may be a unique value sent by the sending system, or possibly a filename in case of file-based integration.

How to configure these parameters

These headers can be populated using the Content Modifier step.

SAP_ApplicationID: Populate the key field from the XPath, camel expression or any supported header types in the content modifier.

SAP_MessageType: Configure the name of the key field you are capturing in SAP_Application ID.

For example;
Ex: National Id : 12312132312

SAP_ApplicationID: //nationalId
SAP_MessageType: National Id
